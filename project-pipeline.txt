GYM & FITNESS SOCIAL MEDIA - PROJECT PIPELINE & DOCUMENTATION
================================================================================

1. PROJECT OVERVIEW
-------------------
A comprehensive social media platform for fitness enthusiasts, featuring user 
authentication, profile management, workout tracking, and community interactions.

2. TECH STACK & ARCHITECTURE
----------------------------
- Frontend: React 18 + TypeScript + Vite
- Backend: Node.js + Express.js
- Database: MongoDB (Atlas) + Mongoose ODM
- Architecture: MVC (Model-View-Controller)
- Auth: JWT (JSON Web Tokens) with localStorage persistence
- Storage: Cloudinary (for image uploads)

3. FRONTEND LIBRARIES & DEPENDENCIES
------------------------------------
Core:
- React 18 & TypeScript (Static Typing)
- React Router Dom: Navigation & Protected Routing
- Vite: Build tool & Optimized Dev Server
- Axios: Centralized API Service Layer

Styling & UI:
- Tailwind CSS: Utility-first styling
- Framer Motion: Fluid animations
- Lucide React: Iconography
- clsx & tailwind-merge: Dynamic class management

Visualization:
- Recharts: Interactive charts for fitness tracking

4. BACKEND LIBRARIES & DEPENDENCIES
-----------------------------------
Core:
- Express: Fast, unopinionated web framework
- Mongoose: Schema-based MongoDB modeling
- Dotenv: Environment variable security

Security & Auth:
- Jsonwebtoken (JWT): Secure stateless authentication
- Bcryptjs: Industry-standard password hashing
- Helmet: HTTP security headers
- Cors: Cross-Origin Resource Sharing
- Morgan: Request logging for development

File Handling:
- Multer & Multer-Storage-Cloudinary: Secure cloud image uploads
- Cloudinary SDK: Cloud-based media management

Validation:
- Express-Validator: Robust request body validation

5. PROJECT STRUCTURE (REORGANIZED)
--------------------
Gym - social media/
├── server/                 # Backend (Node.js/Express)
│   ├── config/             # DB & Cloudinary configurations
│   ├── controllers/        # Request handling logic
│   ├── models/             # Mongoose schemas (User, Workout, Post)
│   ├── routes/             # API endpoints (Auth, Users, Workouts, Posts)
│   ├── middleware/         # Auth Guard & Validation
│   ├── utils/              # Upload helpers
│   └── seed.js             # Automated sample data script
├── src/                    # Frontend (React/TS)
│   ├── assets/             # Static files (images, SVGs)
│   ├── components/         # UI (Reusable) & Layout (Navbar, Sidebar)
│   ├── context/            # AuthContext (Global state)
│   ├── hooks/              # Custom React hooks
│   ├── pages/              # View components (Lazy loaded)
│   ├── services/           # Centralized API layer (api.ts)
│   └── utils/              # Helper functions (utils.ts)
└── package.json            # Root & Application dependencies

6. PERFORMANCE & CODE QUALITY
----------------------------
- Route Lazy Loading: Implemented React.lazy/Suspense for optimal bundle splitting.
- TypeScript Migration: API layer and key components fully typed for stability.
- Memory Optimization: Integrated useCallback/useMemo in high-frequency components.
- Repository Cleanup: Removed redundant Vite templates (temp-app) and build artifacts (dist).
- Robust Error Handling: Centralized backend error middleware and frontend toast-style alerts.

7. API PIPELINE FLOW
--------------------
Client Request -> Router -> Middleware (Auth/Validation) -> Controller -> Model -> MongoDB
MongoDB Response -> Model -> Controller -> Client Response
